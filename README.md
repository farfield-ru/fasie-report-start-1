# Промежуточный научно-технический отчёт по гранту «Старт-1» (ФСИ)

Тема НИОКР: **«Разработка и тестирование прототипа программного комплекса для
моделирования и симуляции роботов с использованием графических ускорителей»**.

Отчёт за **этап 1** оформлен в LaTeX по **ГОСТ 7.32** (класс
[`G7-32`](https://github.com/latex-g7-32/latex-g7-32)) и собирается Docker-образом
шаблона `latex-g7-32/latex-g7-32`.

> Титульный лист, список исполнителей и реферат формируются в системе «Фонд-М»
> и в данный документ не включаются — он начинается с содержания (в соответствии
> с рекомендациями Фонда по оформлению промежуточного НТО).

## Состав

| Файл / каталог | Назначение |
|---|---|
| `report.tex` | Главный файл отчёта |
| `preamble.tex` | Преамбула + макрос заглушек скриншотов |
| `sections/` | Введение, основная часть (по календарному плану), заключение, список источников |
| `refs.bib` | Библиография (стиль `ugost2008`) |
| `figures/` | Сюда кладутся реальные скриншоты (PNG) |
| `latex-g7-32/` | Подмодуль: шаблон ГОСТ 7.32 + класс `G7-32` + стили ГОСТ |
| `scripts/build.sh` | Команды сборки внутри контейнера |
| `Makefile` | Локальная сборка (образ + компиляция) |
| `.github/workflows/build.yml` | CI: сборка PDF и публикация в GitHub Release |

## Сборка локально

Требуется Docker.

```sh
git clone --recurse-submodules <url> fasie-report-start-1
cd fasie-report-start-1
make            # соберёт образ из latex-g7-32 и скомпилирует build/report.pdf
```

Если репозиторий уже склонирован без подмодулей:

```sh
git submodule update --init --recursive
```

## Сборка в CI

Workflow `Build report PDF` при каждом push в `main`:

1. собирает образ из `latex-g7-32/docker/Dockerfile`;
2. компилирует `build/report.pdf`;
3. загружает PDF как artifact и публикует его в перезаписываемый
   [релиз `latest`](../../releases/tag/latest).

Тег вида `vX.Y` создаёт отдельный версионный релиз с вложенным PDF.

## Скриншоты

В тексте расставлены **заглушки** (обрамлённые области с пояснением, что нужно
заснять, и именем файла вида `figures/<метка>.png`). Чтобы вставить реальное
изображение, положите PNG в `figures/` и замените соответствующий вызов
`\screenshot{...}` на `\includegraphics`.
