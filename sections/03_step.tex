\label{cha:step}

\section{Постановка задачи}

Конструкторская документация на робототехнические изделия чаще всего
поставляется в нейтральном формате STEP (ISO~10303)~\cite{iso10303}, который описывает геометрию
сборки и иерархию деталей, но \emph{не содержит} сведений о подвижных
соединениях (сопряжениях, mate-связях). Для использования такой сборки в
симуляции необходимо восстановить её кинематическую схему: дерево звеньев и
суставов с их типами и осями. Задача решается в репозитории \texttt{kinechain}
(язык Python, геометрическое ядро OpenCASCADE~\cite{opencascade} через пакет \texttt{cadquery-ocp}).

\section{Алгоритм}

Восстановление кинематики выполняется как конвейер из шести шагов:

\begin{enumerate}
  \item \textbf{Загрузка STEP} с сохранением иерархии сборки и мировых
        преобразований деталей (OCCT/XCAF).
  \item \textbf{Извлечение аналитических поверхностей} каждой детали ---
        плоскостей и цилиндров --- с параметрами: для плоскости — точка и нормаль,
        для цилиндра — ось, точка на оси и радиус.
  \item \textbf{Обнаружение контактов} пар поверхностей: широкая фаза отбрасывает
        заведомо непересекающиеся детали по перекрытию ограничивающих
        параллелепипедов (AABB); узкая фаза проверяет геометрическую
        совместность (соосность цилиндров, совпадение плоскостей и т.\,п.).
  \item \textbf{Распознавание суставов} по признакам контактов: соосные цилиндры
        пары деталей дают вращательный (\texttt{revolute}) сустав; совмещённая
        призматическая пара --- поступательный (\texttt{prismatic}); жёстко
        прилегающие плоскости без вращательной свободы --- фиксированное
        (\texttt{fixed}) соединение.
  \item \textbf{Построение кинематического дерева}; обнаруженные циклы
        (замкнутые кинематические цепи) разрываются с введением замыкающих
        ограничений (loop constraints).
  \item \textbf{Экспорт модели} в формат MJCF (MuJoCo); экспорт в SDF использует
        то же промежуточное представление.
\end{enumerate}

Признак соосности двух цилиндров с осями $\mathbf{a}_1,\mathbf{a}_2$ и точками на
осях $\mathbf{p}_1,\mathbf{p}_2$ формализуется как одновременное выполнение
условий параллельности осей и совпадения прямых:
\begin{equation}
  \lVert \mathbf{a}_1 \times \mathbf{a}_2 \rVert < \varepsilon_{\angle},
  \qquad
  \bigl\lVert (\mathbf{p}_2 - \mathbf{p}_1) \times \mathbf{a}_1 \bigr\rVert
  < \varepsilon_{d},
  \label{eq:coaxial}
\end{equation}
где $\varepsilon_{\angle}$ и $\varepsilon_{d}$ --- угловой и линейный допуски.
При совпадении радиусов в пределах допуска такая пара трактуется как
цилиндрическое сопряжение и порождает вращательный сустав с осью $\mathbf{a}_1$.

\begin{lstlisting}[language=Python,caption={Распознавание вращательного сустава по соосным цилиндрам (схематично)},label=lst:kinechain]
def detect_joint(part_a, part_b, contacts):
    for sa, sb in contacts:                  # пары контактирующих поверхностей
        if sa.kind == "cylinder" and sb.kind == "cylinder":
            if coaxial(sa, sb) and abs(sa.radius - sb.radius) < EPS_R:
                return Joint(kind="revolute", axis=sa.axis,
                             origin=sa.point, parent=part_a, child=part_b)
    if mating_planes(contacts):              # прилегающие плоскости без свободы
        return Joint(kind="fixed", parent=part_a, child=part_b)
    return None
\end{lstlisting}

\section{Результаты}

Реализован сквозной конвейер построения кинематической схемы из STEP без явной
информации о сопряжениях. На текущем этапе отработан базовый сценарий:
обнаружение вращательного сустава в шарнирной паре деталей и экспорт работающей
модели MJCF. Подтверждена применимость предложенного подхода --- распознавания
суставов по аналитическим признакам контактов поверхностей --- для
автоматической подготовки моделей к симуляции.
